# 前端控制台（Dashboard）傻瓜操作指南

## 这是什么

控制台是猎手-工匠(Craftsman) 服务的可视化管理页面。你可以在浏览器里查看自动发现-生成-上架的全流程状态。

## 如何打开

```powershell
# 1. 先启动 Craftsman 服务
cd D:\A\hunter-craftsman\craftsman
python -m craftsman.cli serve

# 2. 打开浏览器，输入
http://127.0.0.1:8791
```

打开后你会看到一个深色背景的页面，叫 **"Hunter Craftsman 控制台"**。

---

## 页面总览

控制台分左右两栏：

```
┌─ 左栏 ───────────────────┬─ 右栏 ──────────────────────┐
│ 系统总览（4 个数字卡片）    │ 当前运行时间线               │
│ 运行任务列表               │ 当前运行审计                 │
│ 发布任务列表               │ 当前发布详情                 │
│                           │ 审计流                      │
└───────────────────────────┴──────────────────────────────┘
```

---

## 逐个面板说明

### 1. 系统总览（左上）

显示 4 个数字和 4 个检查项：

| 指标 | 说明 |
|------|------|
| 运行总数 | 系统启动以来产生的所有 app 生成任务数 |
| 死信任务 | 失败多次后不再自动重试的任务（卡住的任务） |
| 发布总数 | 尝试上架到 Google Play 的次数 |
| Worker 数 | 后台处理任务的线程数 |

下面 4 行是运行环境检查：
- **工作区** — 存放生成代码的目录是否正常
- **回调目录** — Agent 间通信目录是否正常
- **数据库** — SQLite 是否可连接
- **已修复任务** — 本轮自动修复了多少个卡死状态的任务

### 2. 运行任务（左中）

每次 `hunter autopilot` 或 `hunter run` 触发的 app 开发任务都在这里。

每条显示：
- **opportunity_id** — 任务编号
- **状态** — 如"进行中"、"实现完成"、"实现失败"
- **阶段** — 当前到哪一步了（如：自动发现 → 需求分析 → 开始实现 → 打包 → 完成）
- **run_id** — 唯一 ID，跟日志/调试时有用
- **版本 / 尝试次数** — 第几版、重试了几次

**你能做的操作：**
- 点击 **「时间线」** 按钮 → 右栏显示这个任务的详细进度（每个阶段的时间点）
- 点击 **「重新入队」** 按钮 → 把失败/卡住的任务重新扔回队列重试

### 3. 发布任务（左下）

Agent C 的上架任务列表，每个 app 编译+上架过程记录在这里。

每条显示：
- **release_id — 发布编号**
- **状态** — 如"已准备"、"提交中"、"已发布"、"演练完成"、"失败"
- **平台** — 安卓 / iOS

**你能做的操作：**
- 点击 **「进度」** → 弹出一个窗口，显示完整的发布进度时间线（发布准备 → 审批 → 编译 → 上传 → 成功）
- 点击 **「详情」** → 右栏显示当前发布的详细信息
- 点击 **「审批」**（如果开启了人工审批模式）→ 批准或拒绝发布
- 点击 **「提交」** → 通过审批后提交到 Agent C
- 点击 **「重新入队」** → 失败的发布任务重新排队

---

### 4. 右栏 — 当前运行时间线

当你在左侧任务列表点击某个运行的 **「时间线」** 按钮后，这里展示该运行从开始到现在的每个阶段时间点。

```
示例：
  [10:30:22] 自动发现 → 正在搜索 Play 机会
  [10:31:05] 需求分析 → 正在解析 Blueprint
  [10:32:18] 开始实现 → 正在生成 Compose 代码
  [10:35:44] 打包 → 收集产物
  [10:35:50] 完成
```

### 5. 当前运行审计

显示所选运行的操作记录，如：
- "运行任务已入队"
- "同步运行开始"
- "同步运行完成"

### 6. 当前发布详情

当你在左侧发布列表点击某个发布的 **「详情」** 按钮后，这里显示：

- **发布状态**
- **策略检查结果**（是否通过）
- **审批记录**（谁批准的、备注）
- **Play Console 链接**（发布成功时，直接点击跳转到 Play Console）
- **错误详情**（发布失败时，点击展开查看完整错误栈）

### 7. 审计流

所有 Agent 间的交接记录，按时间倒序排列，用于排查问题。

## 常见场景

### 场景 A：想看看 autopilot 跑得怎么样了

1. 打开 http://127.0.0.1:8791
2. 看左上角「运行总数」是否有增加
3. 在「运行任务」列表中找最新那条
4. 点击 **「时间线」** 看它到哪一步了

### 场景 B：发布失败了，想看原因

1. 在左侧「发布任务」列表找到失败的发布
2. 点击 **「详情」** → 右栏显示发布详情
3. 如果有红色 **「错误详情」** 按钮，点击展开看完整错误信息
4. 修复问题后，点击 **「重新入队」** 重新发布

### 场景 C：上架成功了，想去 Play Console 确认

1. 在左侧「发布任务」列表找到已发布的条目
2. 点击 **「详情」** → 右栏出现绿色 **Play Console** 链接
3. 点击直接跳转到 Google Play Console 查看

### 场景 D：任务卡住了，想重试

1. 找到卡住的任务（状态为死信或失败）
2. 点击 **「重新入队」**
3. 后台 worker 会自动捡起来重试

## 自动刷新

页面默认每 4 秒自动刷新一次数据。你可以在顶部右上角：
- 取消勾选 **「自动刷新」** — 停止自动刷新
- 点击 **「刷新」** — 手动刷新一次

---

## 完全没 UI？前端挂了？

如果页面打不开，按顺序检查：

| 问题 | 解决方法 |
|------|---------|
| 浏览器输入地址后无法连接 | 确认 Craftsman 服务在运行：`python -m craftsman.cli serve` |
| 页面打开但全是空白 | 按 F12 打开开发者工具 → Console 看有没有红字报错 |
| 页面显示但所有数据都是"-" | 还没跑过任何 autopilot 或 run，先跑一次 `hunter autopilot` |
| 点击按钮没反应 | 右键 → 刷新页面，或顶部点「刷新」按钮 |

## 和命令行里的内容有关系吗？

**控制台只是看状态的界面**，不会影响系统运行。实际干活的是命令行里的：

```powershell
# 启动服务（必须先开这个）
python -m craftsman.cli serve

# 触发干活
hunter autopilot           # 只发现+生成
hunter autopilot --publish # 发现+生成+上架
```

控制台只负责给你看它们干得怎么样了。
