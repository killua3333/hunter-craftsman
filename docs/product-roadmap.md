# 产品路线图

## 目标

把 Hunter-Craftsman 从工程原型升级为可演示、可操作、可持续迭代的三智能体产品系统。

## 阶段

1. 前端产品化：用户从工作台理解机会发现、需求池和任务进度。
2. Agent A 升级：输出细分领域、痛点、竞品缺口、评分和选择理由。
3. Agent B 升级：稳定生成可操作 Android MVP，而不仅是可编译代码。
4. Agent C 稳定：第一阶段只提交 Google Play internal track。
5. 文档和验收：形成产品说明、技术规格、操作手册和问题排查文档。

## 暂缓内容

商业模式和 production 发布不在本轮实践范围内。系统保留 monetization、price_tier、收益数据入口，后续再接商业验证。

## 验收标准

- 非技术用户能打开 Dashboard 看懂系统在做什么。
- 每个需求都能解释目标用户、痛点和选择理由。
- 生成的 App 至少有一个可操作主功能。
- C 的 dry-run 和真实 internal 上传状态可区分。
